Welcome to the Etosha propject repository!

Etosha aims at building a bridge between the domains of Big-Data and Linked-Data to support integrated data driven research projects.

We create tools for contextualisation, metadata extraction, and integration.

You, too, can contribute to the Etosha data set graph! Our project Wiki collects and connects metadata in a human and also in a machine readable format. Providing an easy access for non-programmers and programmers alike.

The project is sponsored by Atlassian and managed via our public JIRA.

Example: Relevance Studies in Wikipedia

Based on open data, provided bt the Wikimedia Foundation, we calculate characteristic measures about individual Wikipedia pages – REP- Representation Index and TRRI-Time-Dependend Relevance Index. These measures take the neighbourhood of the selected pages into account. Thus results are embedded into the wider context of a page. Using the Etosha project Wiki, we provide references to our raw data, methodology, obtained results. These results are accessible via an API; build in to the SemanticMedia Wiki.

How Etosha Enables the Global Data Village?

The Etosha dataset graph connects multiple linked data sources. It embeds databases and related metadata into non-technical contexts via semantic links. Such an embedding into the global linked data graph allows plausibility checks and supports interpretation as well as comparison of research results while access control stays always in the users hands. Data has not be moved to a public provider - all facts can be published from a private environment.

Etosha can operate as a (metadata)-gateway between emerging datahubs.

Only active linking between raw data, technical or operational metadatdata, and domain knowledge allows a 360° view on any data set. Public access to social media systems and related usage data, such as the Wikipedia click count dataset, is one of fundamental aspects for Computational Social Science. Furthermore, availability of processing resources became mainstream during the last decade, especially since the breakthrough of the entire Hadoop ecosystem and its commercial recognition.

But real insights require more than just numbers or facts. Therefore, the Etosha project provides scalable context management services.

Efficient Communication is a key to success in general. Digital communication tools are widely accepted but there is an obvious lack of standards which allow the growth of a coherent dataset network – which is comparable to Facebook’s social-graph or the CrunchBase business-graph. Such a dataset graph allows context embedding in collaboration environments and overarching cluster dataset discovery in form of a global dataset catalog. This catalog can be considered as one of the driving forces towards a new era of data economy, based on data analysis and data driven business.

We build the Etosha-Graph to arrange, visualise, filter and share dataset context information using descriptors of our DOAx (Description of a x) ontology. Here Etosha follows the path of the DOAP project and applies the concept of interlinked project-life cycle management to context sensitive dataset life cycle management.
